## Architecture & Layout Refinement

This is a refinement pass only.

Do not redesign the visual identity, component architecture, rendering strategy, or business logic. Preserve the current implementation and improve only proportions, spacing, alignment, and organization.

---

## Component Organization

Organize the project by responsibility.

- Components with dedicated styling must keep their React file and all related styles together.
- Components without dedicated styling should remain a single file.
- Do not create unnecessary folders or files.
- One component, one responsibility.

---

## Styling Organization

Separate styling by responsibility.

- Keep component-specific styles with their respective components.
- Keep only shared styles globally.
- Separate shared typography, layout, print behavior, design tokens, and global rules into independent shared styles.
- Never mix component-specific styles with shared styles.
- Eliminate duplicate styles and selectors.

---

## A4 Page

Every printable page must maintain an exact **210 / 297** aspect ratio.

- Remove any `max-width` constraint.
- Size the page from its aspect ratio only.
- Allow the parent container to scale the page uniformly.
- Never distort or independently scale the page.
- Screen and print output must remain visually identical.

---

## Safe Area

Replace arbitrary page padding with a border-aware safe area.

Maintain two independent spacing zones:

- **Outer Safe Area:** Viewport → Border
- **Inner Safe Area:** Border → Content

Requirements:

- Equal left, right, and top spacing.
- Bottom spacing adjusted only for the bottom cartouche.
- Padding must be derived from the ornamental border, not arbitrary values.
- Decorative corner ornaments define the minimum content boundary.
- Content must never enter the decorative corner or ornament area.

---

## Layout Refinement

Improve only:

- proportions
- spacing
- padding
- alignment
- visual rhythm
- optical balance

Do not redesign components or replace the current implementation strategy.

---

## Navigation

Keep the existing navigation component and logic.

Only reposition it.

- Never overlay the printable page.
- Never cover the border or bottom ornament.
- Place it below the page with balanced spacing.

---

## Code Quality

- Reuse existing implementations before creating new ones.
- Refactor instead of duplicating.
- Separate UI, styling, layout, and logic by responsibility.
- Keep every module focused on a single purpose.
- Remove duplicate code, duplicate styles, and unnecessary abstractions.
